# Jira Workflow Simulation

This project demonstrates a sample Agile QA workflow.

## User Story

TASK-101: Create Task Endpoint

As a user
I want to create tasks through the API
so that I can manage my work items.

### Acceptance Criteria

- API accepts valid task data
- Response returns HTTP 201
- Task is saved successfully
- Invalid requests return proper errors

---

## QA Process

1. Review User Story
2. Create Test Cases
3. Execute API Tests
4. Log Defects
5. Retest Bug Fixes
6. Close Story
